# RehaScope - 要件定義書（SDD）

**バージョン：** 1.0
**作成日：** 2026-03-15
**対象：** 理学療法士向け動作分析Webアプリ

---

## 1. サービス概要

### 1.1 アプリ名
**RehaScope**

### 1.2 目的
理学療法士が患者の動作（立ち上がり・歩行・バランス）をスマホ・タブレットで撮影し、MediaPipeを活用した骨格推定により関節角度・重心偏位を自動計測。介入前後の変化を定量的に可視化することで、臨床評価の質と効率を向上させる。

### 1.3 ターゲットユーザー
- 職場（単一施設）の理学療法士 数名
- メインデバイス：iPad（タブレット横持ち）

### 1.4 解決する課題
- 動作分析が目視・主観的評価に依存している
- 介入前後の変化を数値で示す手段がない
- 専用機器なしに関節角度を客観的に計測したい

## 2. 機能要件

### 2.1 認証
| 機能 | 詳細 |
|------|------|
| パスワード認証 | 起動時に職場共通パスワードを入力。正しい場合のみアプリに入れる |
| パスワード管理 | Vercel環境変数（`NEXT_PUBLIC_APP_PASSWORD`）で管理 |
| セッション | ブラウザセッション中は再入力不要 |

### 2.2 動作種類選択
| 動作種類 | 説明 |
|---------|------|
| 立ち上がり | 椅子座位→立位の動作 |
| 歩行 | 歩行動作 |
| バランス・静止立位 | 静止立位でのバランス評価 |

### 2.3 動画入力
| 機能 | 詳細 |
|------|------|
| 撮影面選択 | 前額面のみ / 矢状面のみ / 両方 の3択 |
| 入力方法 | カメラ撮影（リアルタイム録画）またはファイルアップロード |
| 対象動画 | Before（介入前）/ After（介入後）各1本ずつ × 選択面数 |
| 対応形式 | MP4, MOV, WebM |
| 動画保存 | サーバーへの送信・保存なし。ブラウザ内メモリのみで処理 |

### 2.4 骨格推定・分析
| 機能 | 詳細 |
|------|------|
| エンジン | MediaPipe Pose（Web版・ブラウザ完結） |
| 解析対象 | 全フレームを自動解析 |
| 計測関節 | 股関節・膝関節・足関節（屈曲伸展）・体幹傾斜角（前後左右） |
| 重心推定 | 身体重心の前後左右偏位角度 + 足底上の重心位置プロット |

### 2.5 比較表示画面
| 機能 | 詳細 |
|------|------|
| タブ切替 | 前額面タブ / 矢状面タブ（両面選択時のみ） |
| 動画表示 | Before（左・青枠）/ After（右・橙枠）を左右並列同時再生 |
| グラフ | 全フレームの関節角度時系列グラフ（Before：青線 / After：橙線） |
| 重心プロット | 足底アウトライン上に重心位置をアニメーション表示 |

### 2.6 フレーム指定スナップショット
| 機能 | 詳細 |
|------|------|
| フレーム選択 | 各動画下部のスクラバーで任意フレームを手動指定 |
| 骨格オーバーレイ | 選択フレームにスティックフィギュア + 関節角度ラベル表示 |
| 比較テーブル | Before / After / 差分（Δ）の角度数値を表形式で表示 |

### 2.7 出力
| 機能 | 詳細 |
|------|------|
| PDF出力 | 比較画面のスクリーンキャプチャ + 角度数値サマリーをPDF化 |
| CSVエクスポート | 全フレームの関節角度データをCSV出力 |
| ファイル命名 | `YYYYMMDD_動作種類_撮影面.pdf/csv`（例：`20260315_standing_sagittal.pdf`） |

### 2.8 PWA・オフライン対応
| 機能 | 詳細 |
|------|------|
| PWA | ホーム画面へのインストール対応 |
| オフライン | 初回ネット接続後、Service Workerがアプリ + MediaPipeモデルをキャッシュ |
| 以降の利用 | インターネット未接続でも全機能利用可能 |

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| プライバシー | 動画・患者データをサーバーに送信しない |
| 患者情報 | アプリ内に一切保存しない |
| 対応デバイス | iPad（横持ち）メイン、スマホ（縦持ち）補助対応 |
| ブラウザ | Chrome / Safari 最新版 |
| パフォーマンス | MediaPipe処理：30fps相当でリアルタイム表示を目標 |

## 4. 画面一覧

| 画面ID | 画面名 | 説明 |
|--------|--------|------|
| SCR-01 | パスワード認証画面 | アプリ起動時に表示 |
| SCR-02 | ホーム・動作選択画面 | 動作種類を選択するトップ画面 |
| SCR-03 | 動画入力画面 | 撮影面選択 + Before/After動画入力 |
| SCR-04 | 分析・比較画面 | タブ切替 + 動画並列 + グラフ + 重心 |
| SCR-05 | フレームスナップショット画面 | 任意フレームの角度詳細比較 |

---

## 5. データフロー

```
[ユーザー操作]
  ↓ 動画選択/撮影
[ブラウザ内メモリ（Blob URL）]
  ↓ MediaPipe Pose（WebAssembly）
[関節座標データ（フレームごと）]
  ↓ 角度計算・重心計算
[表示用データ（React State）]
  ↓ jsPDF / CSVエクスポート
[端末ダウンロード]
※ サーバーへのデータ送信なし
```

## 6. 技術スタック

| レイヤー | 技術 |
|---------|------|
| フレームワーク | Next.js 14（App Router）+ TypeScript |
| スタイリング | Tailwind CSS |
| 動作分析 | @mediapipe/pose（Web版） |
| PWA | next-pwa |
| PDF生成 | jsPDF + html2canvas |
| CSV生成 | ネイティブ JavaScript |
| テスト | Jest + React Testing Library |
| デプロイ | Vercel |
| DB/バックエンド | なし（完全フロントエンドアプリ） |
